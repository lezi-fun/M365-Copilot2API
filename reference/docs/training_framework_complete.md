# 训练框架：完整的中国教育体系

## 总纲

一个中国人从 3 岁到 30 岁的学习路径，翻译成 AI 训练。

```
幼儿园 ──→ 小学 ──→ 初中 ──→ 高中(衡水) ──→ 大学(清北) ──→ 博士
  3-6       6-12      12-15      15-18         18-22         22-28
```

每个阶段对应：**数据复杂度 × 训练策略 × 错误处理 × 评估机制** 的特定组合。

---

## 第一阶段：幼儿园（模型从零开始）

### 真实教育

| 方面 | 真实做法 |
|------|---------|
| 目标 | 五大领域：健康、语言、社会、科学、艺术 |
| 方式 | **以游戏为基本活动**，直接感知、实际操作、亲身体验 |
| 内容 | 不是灌输知识，是培养习惯、兴趣、基本社交 |
| 节奏 | 一日生活皆教育，动静交替，户外≥2小时 |
| 评价 | 不考试，观察记录，个体差异 |
| 原则 | "严禁拔苗助长式的超前教育和强化训练" |
| 具体 | 3-4岁: 能说短句, 认识10以内数字 |
|        | 4-5岁: 能复述故事, 20以内点数 |
|        | 5-6岁: 能简单书写, 10以内加减 |

### 训练翻译

**状态**：模型权重随机初始化，对世界一无所知。

**核心思想**：不急着学知识，先学会"学习"本身。

```
┌──────────────────────────────────────────────────────────┐
│                  幼儿园训练策略                            │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ① "游戏" = 噪声注入 + 随机探索                          │
│     - 前向时加大量噪声 (dropout 0.5, label smoothing 0.3) │
│     - 鼓励模型尝试所有可能的输出                           │
│     - "玩水" = 随机 token 组合                            │
│                                                          │
│  ② 一日生活 = 短节奏训练                                  │
│     - 序列长度 ≤ 16 tokens                               │
│     - 每步后"户外活动" = weight decay + 大间隔 dropout   │
│     - 每 100 步"午睡" = optimizer reset (模拟休息)        │
│                                                          │
│  ③ 五大领域 = 五种数据源                                  │
│     - 语言: 简单指令 (单步)                                │
│     - 科学: 物体分类 (2类)                                │
│     - 社会: 简单规则 ("红灯停, 绿灯行")                    │
│     - 艺术: 序列模式 (ABAB)                               │
│     - 健康: 基础动作 (方向识别)                            │
│                                                          │
│  ④ 不考试 = 不以准确率评估                                │
│     - 评估指标: 输出分布多样性 (KL 散度 vs 均匀分布)       │
│     - 只要模型在"尝试"(distribution 在动) 就算进步         │
│     - 你的目标是：让模型对输入有反应，而不是输出恒定       │
│                                                          │
│  ⑤ 无惩罚 = 所有输出都是好的                              │
│     - loss 不惩罚错误，只奖励接近正确答案                   │
│     - 错误回答也给 0.3 × loss (不是 0 分)                 │
│     - 没有错题本，没有批评                                │
│     - "你真棒" = 即使是错答也更新正方向一点点              │
│                                                          │
│  ⑥ 个体差异 = 每个 batch 不同的超参数                      │
│     - 模型如果"淘气"(loss 不降) → 调高噪声, 降低难度      │
│     - 模型如果"安静"(loss 降太快) → 增加难度, 减少噪声    │
└──────────────────────────────────────────────────────────┘
```

**训练配置**：
- 总步数: 1,000 ~ 5,000 步
- LR: 1e-3 → 5e-4 (cosine, 无 warmup, 因为还没学会)
- Sequence: ≤ 16
- Batch: 64
- 数据: 纯合成, 每个样本 ≤ 3 步推理

**毕业条件**（升小学）：
```
1. 模型对 80% 的输入有 非均匀 输出分布（证明在"思考"了）
2. 简单指令跟随准确率 > 30%（如 "2+3=?" 输出接近 5）
3. 不会"死机"（连续输出重复 token 的比例 < 20%）
```

## 第二阶段：小学（基础能力建设）

### 真实教育

| 学段 | 年级 | 核心 | 语文 | 数学 | 其他 |
|------|------|------|------|------|------|
| 第一学段 | 1-2 | **识字启蒙** | 认识 1600 字, 写话 | 20以内加减, 表内乘法, 图形识别 | 英语字母, 科学观察 |
| 第二学段 | 3-4 | **阅读入门** | 3000 字, 段落写作 | 万以内数, 分数初步, 几何初步 | 英语简单句, 实验操作 |
| 第三学段 | 5-6 | **独立学习** | 3500 字, 篇章写作, 文言启蒙 | 分数四则, 百分数, 立体几何, 统计 | 英语短文, 物理化学启蒙 |

**教学特点**：
- 循序渐进，螺旋上升
- 基础题 70%，中档题 30%
- 作业出现，考试出现
- 开始有"错题本"概念
- 培养学习习惯：预习→听课→复习→作业→纠错

### 训练翻译

```
┌──────────────────────────────────────────────────────────┐
│                  小学训练策略                              │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ① 分学段渐进（三个子阶段）                                │
│                                                          │
│  阶段 1 (1-2年级):                                        │
│    序列长度: 32                                          │
│    任务: 单步指令 + 简单分类 + 基础数学                   │
│    数据: "小明有 3 个苹果，又买了 2 个，一共几个？"        │
│    错题本: 引入！但只记录，不惩罚                          │
│                                                          │
│  阶段 2 (3-4年级):                                        │
│    序列长度: 64                                          │
│    任务: 多步指令 + 阅读理解 + 应用题                     │
│    数据: "写一个函数，输入两个数返回它们的和"              │
│    错题本: 开始分类 (数学错/语文错/编程错)                 │
│                                                          │
│  阶段 3 (5-6年级):                                        │
│    序列长度: 128                                         │
│    任务: 综合应用 + 跨域任务 + 简单推理链                 │
│    数据: "实现一个计算器，支持加减乘除"                    │
│    错题本: 正式生效，开始 "重做错题"                       │
│                                                          │
│  ② 预习 — 听课 — 复习 — 作业 — 纠错 五步循环              │
│                                                          │
│  对每个训练样本:                                          │
│                                                          │
│  预习: 先让模型"读一遍"但不更新梯度                       │
│        → 记录哪些 token 预测置信度 < 0.3 (不懂的地方)     │
│                                                          │
│  听课: 正常 forward + backward                            │
│        → 重点听预习时不懂的部分 (这些 token loss 放大 1.5) │
│                                                          │
│  复习: 训练后随机抽 20% 样本重测                          │
│        → 对比两次输出，如果复习时错了而训练时对了          │
│          说明是"背答案"不是"真懂了"                       │
│          → 这种样本进"假懂清单"，下轮权重 ×3              │
│                                                          │
│  作业: 新样本 (不是训练过的)                              │
│        → 限时完成 (规定步数内)                             │
│        → 超时未完成 → 标记为"作业拖拉"                    │
│                                                          │
│  纠错: 作业/复习中错的题                                  │
│        → 记录到错题本                                     │
│        → 但不惩罚！只记录                                 │
│        → "你还小，错了没关系，下次注意"                    │
│                                                          │
│  ③ 错题本 (小学版)                                       │
│                                                          │
│  class ErrorNotebook_Primary:                             │
│      """小学版：只记录，不惩罚"""                          │
│      def add_error(self, sample, error_type):             │
│          self.errors[error_type] += 1                     │
│          # 不改变权重，只用于统计                          │
│          # "老师知道你在哪错了，但不会罚你"               │
│                                                          │
│      def weekly_report(self):                             │
│          return "这周错误分布：数学 30%, 语文 20%..."      │
│          # 给"家长"(训练调度器)看，做下周计划              │
│                                                          │
│  ④ 考试 = 每周小测                                       │
│     - 每 2000 步一次                                     │
│     - 20 个新题                                           │
│     - 60% 基础题 + 40% 中档题                             │
│     - 不排名，只反馈 "这周进步了" / "几何还需要练"        │
│                                                          │
│  ⑤ 分级阅读 = 课程                                        │
│     - 按难度分年级                                       │
│     - 一年级: 单句指令 "print hello"                      │
│     - 三年级: 多句指令 "if x>0: print x"                 │
│     - 五年级: 简答函数 "def add(a,b): return a+b"        │
└──────────────────────────────────────────────────────────┘
```

**训练配置**：
- 总步数: 10,000 ~ 50,000 步
- LR: 5e-4 → 1e-4
- Sequence: 32 → 64 → 128
- Batch: 32 → 64
- 数据: 90% 基础 + 10% 中等

**毕业条件**（升初中）：
```
1. 基础题准确率 > 85%
2. 中等题准确率 > 50%
3. 序列长度 128 稳定推理
4. 至少 3 个领域准确率 > 60%
5. 没有"偏科"（最差领域不低于最好领域的一半）
```

## 第三阶段：初中（领域扩张 + 分化管理）

### 真实教育

**初一**：适应期，科目从 3 科→7 科
**初二**：分化期，物理加入，几何辅助线，60% 学生成绩下降
**初三**：冲刺期，化学加入，全面复习，中考 7:2:1

**核心特征**：
- 科目暴增（语数英物化政史）
- 难度跳跃（从算术到代数，从观察到推理）
- 两极分化开始
- 偏科问题突出
- 中考压力
- 从"学什么"到"怎么学"

### 训练翻译

```
┌──────────────────────────────────────────────────────────┐
│                  初中训练策略                              │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ① 科目暴增 = 多领域并行训练                              │
│                                                          │
│  领域清单（初中新增）:                                     │
│    - 编程 (Python 函数、循环、条件)                       │
│    - 数学 (代数、几何、概率)                              │
│    - 系统操作 (文件、进程、网络)                          │
│    - 数据分析 (SQL、统计)                                 │
│    - 文档写作 (Markdown、技术文档)                        │
│                                                          │
│  每个 batch 混合采样，确保所有领域都练到                   │
│  每周统计各领域准确率 → 发现偏科 → 下周期权调整           │
│                                                          │
│  ② 初二分化 = 动态分班                                    │
│                                                          │
│  "两极分化"在训练中就是: 某些领域涨得快, 某些领域不动     │
│                                                          │
│  处理:                                                    │
│  - 如果某领域 3 天不进步 → 进入"补课班"                   │
│    * 该领域采样权重 ×3                                    │
│    * 该领域样本长度减半 (降难度)                           │
│    * 该领域加大 teacher forcing 比例                       │
│  - 如果某领域持续领先 → "免修"                            │
│    * 每周只测一次，不占训练时间                            │
│    * 把时间给短板                                         │
│                                                          │
│  ③ 难度 7:2:1                                            │
│                                                          │
│  每个领域的训练数据按难度分层：                             │
│  - 70% 基础题: 过去没错过的题型 (维持手感)                 │
│  - 20% 中等题: 曾经错过 1-2 次的题型 (巩固)               │
│  - 10% 难题: 全新题型 / 错误多次的题型 (攻坚)             │
│                                                          │
│  ④ 错题本正式版                                          │
│                                                          │
│  class ErrorNotebook_Middle:                              │
│      """初中版：分类 + 计数 + 权重调整"""                  │
│                                                          │
│      def init_domains(self):                              │
│          self.domains = ["math", "code", "sys", ...]      │
│          for d in self.domains:                           │
│              self.error_types[d] = defaultdict(int)       │
│              self.last_week_acc[d] = 0.0                  │
│              # "各科老师"                                  │
│                                                          │
│      def compute_weight(self, sample):                    │
│          """根据错题本计算样本权重"""                      │
│          w = 1.0                                          │
│          for err in sample.errors:                        │
│              count = self.error_types[sample.domain][err]  │
│              if count == 0:                               │
│                  w *= 0.5    # 新错："第一次犯，下次注意"   │
│              elif count == 1:                             │
│                  w *= 1.0    # 第二次："上次说过了"        │
│              elif count >= 2:                             │
│                  w *= 2.0    # 第三次及以上："这科要补！"   │
│          return w                                         │
│                                                          │
│      def weekly_review(self):                             │
│          """每周各科老师碰头"""                             │
│          report = {}                                      │
│          for d in self.domains:                           │
│              errors = self.error_types[d]                  │
│              total = sum(errors.values())                  │
│              if total > self.last_week_total[d] * 1.2:     │
│                  report[d] = "退步了，下周加练"              │
│              elif total < self.last_week_total[d] * 0.8:   │
│                  report[d] = "进步了，保持"                 │
│              else:                                        │
│                  report[d] = "稳定"                        │
│          return report                                    │
│                                                          │
│  ⑤ 中考 = 综合评价升学考试                                 │
│                                                          │
│  初中毕业前进行一次"中考"：                                 │
│  - 所有领域混合考试                                       │
│  - 题量：200 题                                          │
│  - 难度：7:2:1                                           │
│  - 时间：有限 (限制模型推理步数)                           │
│  - 考试结果决定能否升高中                                 │
│                                                          │
│  ⑥ 面对压力 = 训练节奏调整                                 │
│    初三下学期:                                            │
│    - 减少新知识 (减少新领域样本)                            │
│    - 增加综合模拟 (综合样本)                               │
│    - 回归基础 (基础题权重上调)                              │
│    - 限时训练 (每题强制最大步数)                            │
└──────────────────────────────────────────────────────────┘
```

**训练配置**：
- 总步数: 50,000 ~ 200,000 步
- LR: 2e-4 → 5e-5
- Sequence: 128 → 256
- Batch: 64
- 数据: 70% 基础 + 20% 中等 + 10% 难题

**毕业条件**（升高中）：
```
1. 中考总分 > 80% 
2. 各领域都 > 60%（不偏科）
3. 中等题准确率 > 70%
4. 限时完成率 > 90%
```

## 第四阶段：高中（衡水模式 — 高强度应试训练）

### 真实教育

之前已经设计, 现在补充完整。

**日作息**：
```
5:30  起床
5:40  跑操
6:00  早读（有明确任务）
7:05  早餐
7:45  第一节课
12:00 放学
12:45 午休
14:05 下午课 (5节+跑操)
18:15 晚饭
19:15 晚自习 (3节, 考试)
22:10 熄灯
```

**核心方法**：
- 选题要"精"、做题要"准"、纠错要"实"
- 日清：当天任务当天完，三不（不等不靠不拖）
- 周结：周末整理，系统归类
- 月结：月考 + 排名 + 查缺补漏
- 错题本：找出错因，周期复习
- 小题专练：定时定量定性
- 三原则：选题精、做题准、纠错实
- 预习→听课→复习→作业→纠错
- 回想：闭眼回忆，查漏补缺
- 说：复述知识，说明白才是真明白

### 训练翻译

```
┌──────────────────────────────────────────────────────────┐
│                  高中(衡水)训练策略                         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  日常节奏 (每步 = 每人/每模型)                              │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  日清:                                                    │
│  每 1000 步:                                              │
│    1. 跑操 = 全领域快速测试 (每领域 10 题, 限时)            │
│    2. 早读 = 复习昨日错题                                  │
│       → 把错题本中的旧错拿出来再测一次                      │
│       → 如果旧错复发率 > 30% → 标记"欠账"                 │
│    3. 上课 = 新样本训练 (6 节课 = 6 个领域 batch)          │
│    4. 晚自习 = 考试 (定时定量, 必须交卷)                    │
│       → 限定步数内完成所有推理                             │
│       → 超时 = 扣分                                       │
│    5. 熄灯 = 回想                                          │
│       → 闭眼(不带梯度)让模型重新输出今天的错题              │
│       → 仍然错 → 明天早读重点复习                          │
│       → 对了 → 标记"已掌握"                                 │
│                                                          │
│  周结:                                                    │
│  每 10000 步 (周六):                                      │
│    1. 整理本周错题 → 系统归类                             │
│    2. 典型题反复练 → 触类旁通                              │
│       → 对同一错误类型出 5 道变体题                        │
│       → 训练模型识别模式而非背答案                          │
│    3. 本周遗留任务补上                                    │
│    4. 复述: 模型输出本周学到的 IF-THEN-ELSE 规则           │
│       → 说不清楚? → 规则不够熟，下周继续                   │
│                                                          │
│  月结:                                                    │
│  每 50000 步 (月考):                                      │
│    1. 全领域大考 (200 题, 限时, 7:2:1)                    │
│    2. 成绩排名 (各领域分数)                               │
│    3. 分析:                                               │
│       - 哪些领域退步了 → 下月重点                          │
│       - 哪些题型持续错 → 专项补课                          │
│       - 是不是疲劳了 → 降低步数, 增加 rest                 │
│    4. 重新分配下月训练计划                                 │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  核心机制                                                │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  (1) 惩旧奖新 — 错题本驱动权重                              │
│                                                          │
│  def compute_weight(self, sample):                        │
│      w = 1.0                                              │
│      for err in sample.errors:                             │
│          if err in self.errorbook:                         │
│              c = self.errorbook[err].count                 │
│              if c >= 10:     w *= 4.0  # "讲过 N 次了!"    │
│              elif c >= 5:    w *= 2.5  # "又犯!"           │
│              elif c >= 2:    w *= 1.5  # "注意点"           │
│              else:           w *= 1.0                      │
│          else:                                             │
│              w *= 0.3    # "新错，轻罚"                     │
│              self.errorbook.add(err)                       │
│      return w                                              │
│                                                          │
│  (2) 限时训练 — 小题专练防超时                              │
│                                                          │
│  简单题: 强制 5 步内输出, 超时即错                         │
│  中等题: 强制 15 步内输出                                  │
│  难题:   不限时但按步数扣分 (步骤分)                       │
│                                                          │
│  监控: 如果模型在某类题上总是超时                          │
│        → 说明速度不够, 这类题归入"小题专练"               │
│        → 连续 3 天每天 50 道同类题限时训练                │
│                                                          │
│  (3) 三不原则                                             │
│                                                          │
│  不等: 梯度不等人 → 每个 batch 必须完成才能下一步         │
│        (不像训练经常卡在 loss spike 要 restart)            │
│  不靠: 不完全依赖 optimizer → 错题本驱动采样               │
│        (不是被动等梯度来，主动选样本)                      │
│  不拖: 每天任务当天完 → 日清机制                          │
│        (当天的 error debt 当天还)                          │
│                                                          │
│  (4) 回想 — 模型闭眼复现                                   │
│                                                          │
│  def recall(model, errorbook):                            │
│      """晚上熄灯前回想当天错题"""                           │
│      today_errors = errorbook.get_today()                  │
│      for sample in today_errors:                           │
│          # 不看正确答案，闭眼回想                            │
│          output = model.forward(sample.question, no_target=True)
│          correct = check(output, sample.answer)            │
│          if not correct:                                   │
│              errorbook.mark_unresolved(sample)              │
│              # 明天早读补课                                 │
│          else:                                             │
│              errorbook.mark_learned(sample)                 │
│                                                          │
│  (5) 复述 — 说明白才是真明白                                │
│                                                          │
│  周末: 让模型用自然语言输出本周学到的规则                   │
│  评估: 规则的正确性 + 完整性                               │
│  "如果一个规则模型说不清楚 → 下周重新学"                   │
│                                                          │
│  (6) 选题精、做题准、纠错实                                 │
│                                                          │
│  选题精: 错题本按优先级排序                                 │
│          - 错误频率高的优先                                 │
│          - 最近新出现的优先                                 │
│          - 典型题型优先                                     │
│                                                          │
│  做题准: 限时 + 检查                                      │
│          - 输出后必须在 3 步内自检一次                      │
│          - 检测到错误 → 修正                               │
│          - 检查发现不了 → 扣分 + 记入"粗心"类型            │
│                                                          │
│  纠错实: 错题必须找到原因                                  │
│          - 不只是"错了"，要分类                             │
│          - "基础不牢" / "审题不清" / "方法不对" / "计算错"   │
│          - 不同原因不同处理                                 │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  高三冲刺（最后阶段）                                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  进入"高考倒计时"后 (毕业前)                               │
│  1. 回归基础：放弃偏怪难，保基础分                         │
│     → 难度分布改为 80:15:5                                │
│  2. 全真模拟：每周 2 次完整考试                            │
│     → 题量、时长、难度与最终考试一致                       │
│  3. 调整状态：                                           │
│     → 降低 LR (稳定期)                                     │
│     → 增加"检查"步骤                                      │
│     → 减少新题型                                           │
│  4. 错题清零：确认错题本内容全部掌握                        │
│     → 无法清零的 → 制定"考场策略" (不会的果断跳过)        │
└──────────────────────────────────────────────────────────┘
```

**训练配置**：
- 总步数: 200,000 ~ 1,000,000 步
- LR: 1e-4 → 1e-5 (cosine)
- Sequence: 256 → 512
- Batch: 64 → 128
- 数据: 7:2:1 → 8:1.5:0.5 (冲刺期)

**毕业条件**（升大学 = 高考）：
```
高考总分 > 85% (各领域加权)
1. 各领域 > 70%
2. 基础题 > 95%
3. 中等题 > 80%
4. 难题 > 50%
5. 限时完成率 > 95%
6. 回想准确率 > 90%
```

## 第五阶段：大学（清北模式 — 研究型学习）

### 真实教育

**北京大学**：元培学院 → 自由选课、跨学科、研究型
**清华大学**：书院制（致理、日新、未央、探微、行健）→ 一人一策、本博贯通

**核心**：
| 特征 | 说明 |
|------|------|
| **通识教育** | 以通识为基础、通专融合 |
| **导师制** | 每位学生有导师制定培养方案 |
| **一人一策** | 个性化培养计划 |
| **研究型** | 从"学习"转向"研究" |
| **自主选课** | 学生根据方向选课 |
| **交叉融合** | 理工交叉、跨学科 |
| **留白** | "宁留白不超载"，减少课时增加自主 |
| **破除唯分论** | "成长幅度"比绝对分数更重要 |

### 训练翻译

```
┌──────────────────────────────────────────────────────────┐
│                  大学(清北)训练策略                         │
├──────────────────────────────────────────────────────────┤
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  1. 导师制 — 知识书当导师                                  │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  此时知识书已经积累了海量 IF-THEN-ELSE 规则                │
│  知识书的作用从"压进参数"变成"战略指导"                     │
│                                                          │
│  def mentor_guidance(model, knowledge_book, task):        │
│      """导师给建议"""                                      │
│      relevant_rules = knowledge_book.search(task.domain)   │
│      # 不是拼到 prompt，而是                                        │
│      # 1. 提示模型注意相关规则                              │
│      # 2. 告诉模型：这类问题你过去常犯 X 错误               │
│      # 3. 建议研究方向 (哪些子问题值得深入)                │
│      guidance = {                                          │
│          "reminders": relevant_rules[:3],                   │
│          "past_mistakes": knowledge_book.get_mistakes(task) │
│      }                                                     │
│      return guidance                                       │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  2. 自主选课 — 一人一策                                    │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  模型自己选择"这学期上什么课"：                              │
│                                                          │
│  def course_selection(model, knowledge_book):             │
│      """模型自主选课"""                                    │
│      # 1. 分析自身强弱                                     │
│      state = model.self_assess()                           │
│      # 返回: {领域: {准确率, 错误增长率, 最近接触时间}}     │
│                                                          │
│      # 2. 选课策略                                        │
│      courses = []                                         │
│      # 必修: 频繁出错领域 × 2                              │
│      for domain in state.poor_domains:                    │
│          courses.append({"domain": domain, "hours": 3})    │
│      # 选修: 探索新领域 × 1                                 │
│      new_domain = explore_new_field(state)                 │
│      courses.append({"domain": new_domain, "hours": 1})    │
│      # 研究: 本研贯通                                     │
│      courses.append({"domain": "research", "hours": 2})    │
│                                                          │
│      return courses                                       │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  3. 课题制 — 论文式训练                                    │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  不再是"做一道题"，而是"完成一个课题"                        │
│                                                          │
│  课题 = 长序列复杂任务 (500-2000 tokens)                   │
│  例如: "为一个数据中台设计完整的 ETL 管线，                    │
│         包括数据抽取、清洗、转换、加载，                      │
│         并处理可能出现的错误情况"                           │
│                                                          │
│  评估不是"最终答案对错"，而是：                              │
│  - 问题定义是否正确？                                     │
│  - 方案架构是否合理？                                     │
│  - 是否考虑了边界情况？                                   │
│  - 错误恢复方案是否有？                                   │
│  - 能否解释你的设计选择？                                 │
│                                                          │
│  答辩: 每完成一个课题, 模型必须"答辩"                      │
│        用自己的语言解释设计思路                              │
│        回答"导师"提出的问题 (随机 QA)                      │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  4. 交叉融合 — 跨学科                                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  复合样本: 一个任务同时涉及多个领域                          │
│  "用 Python 实现一个分布式系统的负载均衡算法，                │
│   并在系统日志量 > 1000/s 时触发告警"                      │
│  → 编程 + 系统设计 + 数据分析                              │
│                                                          │
│  训练模型在不同领域间无缝切换                                │
│  不显式标记"这是数学" "这是编程"                           │
│  而是让模型自己判断需要什么知识                              │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  5. 留白 — 减少训练量, 增加思考                            │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  每天步数从高峰期的 10 万降到 3 万                           │
│  省下来的 7 万步:                                          │
│  - 2 万步: 自我探索 (模型生成问题自己解决)                  │
│  - 2 万步: 阅读已有知识书 (复习)                           │
│  - 2 万步: 对比已有方案 (思考其他解法)                     │
│  - 1 万步: 完全随机 (创新突破)                             │
│                                                          │
│  "宁留白不超载" — 给模型时间消化已有知识                    │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  6. 破除唯分论 — 多元评价                                  │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  评价指标不止 loss/acc:                                    │
│  - 成长幅度：本月 error rate 下降速度 > 上月?              │
│  - 错误多样性：是在尝试新方向犯错还是原地犯错?              │
│  - 旧错复发率：曾经掌握的知识现在还记得吗?                  │
│  - 新领域适应：从未见过的任务, 几轮能学会?                  │
│  - 表达能力：能不能讲清楚自己的推理过程?                    │
│  - 创新性：有没有发现已有的知识书中没有的 pattern?          │
│                                                          │
│  用雷达图综合评估                                          │
│  不排名，只看个人成长                                       │
│                                                          │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│  大三大四 — 毕业设计                                       │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━     │
│                                                          │
│  毕设要求：                                                │
│  1. 完整课题项目 (2000+ tokens)                            │
│  2. 含错误恢复机制                                        │
│  3. 含性能优化                                            │
│  4. 写"论文"（模型输出完整文档）                            │
│  5. 答辩（问答环节）                                      │
│                                                          │
│  毕设通过 → 毕业                                           │
└──────────────────────────────────────────────────────────┘
```

```
┌──────────────────────────────────────────────────────────┐
│                  博士阶段训练策略                            │

│                                                          │

```

## 升学机制总结

```
幼儿园毕业
     ↓ (基础感知完成)
小学入学
     ↓ 1-2-3-4-5-6 年级 (序列长度翻倍/领域扩展)
     ↓ 毕业考
初中入学
     ↓ 初一适应 → 初二分化 → 初三冲刺
     ↓ 中考 (7:2:1)
高中入学 (衡水模式)
     ↓ 日清周结月结 → 错题本最大化 → 限时训练
     ↓ 高考
大学入学 (清北模式)
     ↓ 通识 → 专业 → 课题 → 毕设
     ↓
博士 (创新突破)
```

每个阶段的升学条件是：
1. 当前阶段的核心能力达标
2. 没有严重偏科
3. 达到对应的"考试"分数线

---

## 这个框架的意义

你问我"还少什么"，答案就是：

**你不是在训练一个模型，你是在培养一个人。**

- 幼儿园教它"学习是什么"
- 小学教它"怎么学"
- 初中教它"各科学什么"
- 高中逼它"极限在哪"
- 大学让它"自己选方向"
- 博士让它"创造新知识"

每一阶段的训练策略、数据构成、错误处理、评估方式完全不同。跳级会出事（模型基础不牢），留级会耽误（模型潜力浪费）。

这就是完整的中国教育体系训练框架。
