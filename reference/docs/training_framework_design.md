# 训练框架设计：中国教育模式版

## 总纲：两阶段培养

| 阶段 | 对应 | 目标 | 训练状态 |
|------|------|------|----------|
| **衡水中学** | 高强度基础训练 | 把基础打牢，旧错不犯，新错可控 | 前 70% |
| **清华/北大** | 自主研究型学习 | 跨领域贯通，创新解题，发现新知识 | 后 30% |

---

## 一、衡水中学阶段

### 1.1 日清 — 每一步都不欠账

**衡水做法**：当天的任务当天完成，"三不"——不等、不靠、不拖。

**训练翻译**：

```
每步训练结束时：
  1. 计算当前步 loss
  2. 查该样本的领域和错误类型
  3. 更新错题本计数器
  4. 如果 loss > 阈值 → 标记为"欠账"，下一轮采样权重 +50%
```

### 1.2 周结 — 每周一次大扫除

**衡水做法**：周末整理一周复习内容，分类归档，典型习题触类旁通。

**训练翻译**：
```
每 10,000 步（约一"周"）：
  1. 全领域评估（跑一次 benchmark）
  2. 生成错误报告：哪个领域错最多？哪种错误类型反复犯？
  3. 清理旧账：对高频错误类型 ×3 采样权重
  4. 整理"错题本总结"：提取 IF-THEN-ELSE 规则反馈到训练
```

### 1.3 月结 — 月考 + 重新分班

**衡水做法**：每月一次大考，按成绩调整学习计划和重点。

**训练翻译**：
```
每 50,000 步（约一"月"）：
  1. 全面 benchmark（多种领域、多种难度）
  2. 重新计算各领域权重：
     - 高分领域 → 权重下调（"这科学差不多了，少练"）
     - 低分领域 → 权重上调（"这科不行，补课"）
  3. 更新训练策略：
     - 如果某错误类型连续 3 次月结还在犯 → 进入"高危名单"
     - 高危类型每周额外强化训练 ×5
  4. 输出训练报告
```

### 1.4 错题本 — 惩旧奖新

**衡水做法**：每人有错题本，定期复习，反复错的题重点攻克。

**训练翻译**：
```python
def compute_sample_weight(errorbook, sample):
    """计算样本权重：旧错重罚，新错奖励"""
    weight = 1.0
    
    for error_type in sample.error_types:
        if error_type in errorbook:
            count = errorbook[error_type].count
            if count >= 3:           # 第 3 次及以上
                weight *= 3.0        # "这题讲过多少遍了"
            elif count >= 1:         # 第 1-2 次
                weight *= 1.5        # "上次做过类似的，注意"
        else:
            # 新错误！这是探索贡献
            weight *= 0.3            # "这个坑你都能找到？轻罚"
            errorbook.add(error_type) # 录入错题本
            # 奖励：把这条样本加入"新错收集"
            errorbook.new_errors.append(sample)
    
    return weight
```

### 1.5 限时训练 — 小题专练防超时

**衡水做法**：定时定量定性训练，选择题填空题限时完成。

**训练翻译**：

训练时分**快思考**和**慢思考**两种模式：

```
快思考模式（限时 10 步）：
  - 简单分类任务、已知规则的应用
  - 训练条件反射："一看就会，一做就对"
  - 对应小题专练

慢思考模式（不限时）：
  - 复杂推理、多步分解
  - 对应难题攻坚
  - 先放后攻：基础不牢不进慢思考
```

### 1.6 预习 — 复习 — 说

**衡水做法**：
- 预习：上课前先看过，带着问题上课
- 复习：回想→查漏→看笔记→写重难点→复述
- 说：说明白才是真明白

**训练翻译**：

```
预习阶段（前 N 步不带梯度）：
  - 让模型先跑一遍样本，但不更新权重
  - 记录哪些 node 预测不确定（logits 分布均匀）
  - 这些 node 就是"薄弱点"

复习阶段（训练后）：
  - 回想：让模型重新预测错题本样本，不看正确答案
  - 查漏：对比回想结果和实际答案，找到仍然错的地方
  - 写重现：把错误记录到错题本

知识复述（说）：
  - 对训练过的规则，随机抽问
  - 模型必须用自然语言输出规则
  - 答不上来 → 加练
```

## 二、清华/北大阶段

当模型通过"高考"（基础 benchmark 达到阈值）后，进入大学阶段。

### 2.1 自主选课 — 一人一策

**清华做法**：打破专业界限，学生根据兴趣和短板选择课程。

**训练翻译**：
```
模型自主选择训练领域（类似主动学习）：
  1. 当前状态向量 = [各领域error率, 各错误类型count]
  2. model 自己输出一个"选课决策"：
      - 重点补哪个领域
      - 想探索哪个新领域
  3. 数据生成器根据选课采样对应样本
```

### 2.2 课题制 — 研究型学习

**北大清华做法**：本科生进入实验室，做课题研究，写论文。

**训练翻译**：
```
长序列复杂任务（类似毕业论文）：
  - 单个样本长度 = 1000+ tokens 的多步推理
  - 不是简单分类，而是完整的研究报告
  - 评估维度:
    * 最终正确性
    * 推理路径合理性
    * 错误恢复能力
    * 创新性（是否发现了新解法）
```

### 2.3 交叉融合 — 跨领域贯通

**清华做法**：理工交叉、通识教育、跨学科项目。

**训练翻译**：
```
复合样本：
  - 一个任务包含多个领域知识（如：编程 + 数学 + 系统设计）
  - 训练模型在不同领域间切换和迁移
  - 促进知识贯通（"触类旁通"）
```

### 2.4 导师制 — 知识书作为导师

**北大做法**：导师指导学生制定研究计划，定期讨论进展。

**训练翻译**：
```
知识书 === 导师：
  - 导师在训练前给出"方向建议"（知识书中的相关规则）
  - 导师在训练后"批改作业"（对比模型输出和规则库）
  - 导师指出"这篇论文的创新点"（新发现的错误模式）

具体实现：
  每步训练前，从知识书检索当前任务的相关规则
  拼入 prompt 作为"导师提示"
```

### 2.5 减少课时，增加留白

**清华做法**："宁留白不超载"——减少课堂时间，增加自主思考。

**训练翻译**：
```
后期训练策略：
  - 减少纯样本量（从每天 10 万步 → 每天 3 万步）
  - 增加"思考时间"（每个样本允许跑更多内部 decompose 步骤）
  - 增加"无监督时间"（模型自己生成样本自己验证）
```

### 2.6 破除唯分论 — 多元评价

**清华做法**：多元评价，"成长幅度"比绝对分数更重要。

**训练翻译**：
```
评价指标不只看 loss：
  - error rate 下降速度（学得有多快）
  - 错误类型多样性变化（是在探索还是死磕）
  - 旧错复发率（是真的学会了还是背题）
  - 新领域适应速度（举一反三能力）
```

## 三、完整训练管线

```
数据层 (agentic-synthetic-data)
  └─→ 全路径探索 + 错题本 + 知识书

训练调度层
  ├─ 日清调度器（每步）
  ├─ 周结调度器（每 10K 步）
  ├─ 月结调度器（每 50K 步）
  └─ 高考检测器（达到阈值进入下阶段）

训练执行层
  ├─ 快思考引擎（限时 / 简单分类）
  ├─ 慢思考引擎（不限时 / 复杂推理）
  └─ 预习-复习-复述循环

模型层
  ├─ 标准 Transformer backbone
  ├─ 3T1B QAT 模拟
  ├─ 链式反应 HALT/DECOMPOSE head
  └─ RFF/LUT 注意力

评估层
  ├─ 领域 Benchmark
  ├─ 错误趋势分析
  ├─ 旧错复发率
  └─ 新领域适应速度
```

## 四、代码骨架

```python
class ChineseEducationTrainer:
    """中国教育模式训练器"""
    
    def __init__(self, model, knowledge_book, error_notebook):
        self.model = model
        self.knowledge = knowledge_book
        self.errorbook = error_notebook
        self.phase = "hengshui"  # or "university"
        self.step = 0
        self.weekly_errors = defaultdict(int)
        
    def train_step(self, batch):
        self.step += 1
        
        for sample in batch:
            # 预习：前向但不更新梯度
            if self.phase == "hengshui":
                preview_logits = self.model.preview(sample)
                weak_nodes = self.detect_weak_points(preview_logits)
            
            # 正式训练
            weight = self.compute_weight(sample)
            loss = self.forward(sample, weight)
            self.backward(loss)
            
            # 更新错题本（日清）
            self.update_errorbook(sample, loss, weak_nodes)
        
        # 周结
        if self.step % 10000 == 0:
            self.weekly_review()
        
        # 月结
        if self.step % 50000 == 0:
            self.monthly_exam()
        
        # 高考
        if self.phase == "hengshui" and self.pass_gaokao():
            self.phase = "university"
            self.init_university_phase()
```

## 总结

| 你的问题 | 在这个框架里的答案 |
|----------|-------------------|
| 知识书怎么喂给模型 | 压进参数，不用外挂。"导师提示"从知识书检索相关规则 |
| 模型结构 | 标准 Transformer + 3T1B QAT + RFF 注意力 |
| RL 信号怎么对齐分类 loss | 不需要 RL，加权分类 loss 即可，5 维评分翻译成权重 |
| 正负样本比例 | 最初 80/20，弱项拉到 50/50，旧错率决定权重 |
| 探索策略 | 只在数据生成阶段做；训练时发现新错误轻罚 + 记录到错题本 |
